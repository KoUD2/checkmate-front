## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser→nginx→nestjs | Request crosses internet boundary. JWT cookie/bearer carries identity claims. Cookies are httpOnly via existing auth middleware. |
| nestjs-controller→service | Controller delegates to service after `ValidationPipe(whitelist: true)` strips unknown fields. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-10 | Spoofing | Unauthenticated request to admin/variants endpoints | mitigate | `@UseGuards(JwtAuthGuard, RolesGuard)` at class level on VariantsAdminController. Verified by spec test "RolesGuard blocks USER role on @Roles(Role.ADMIN) routes". |
| T-03-11 | Elevation of Privilege | USER-role JWT submitting POST /admin/variants | mitigate | `@Roles(Role.ADMIN)` enforced by RolesGuard reading reflector metadata. Class-level decorator means every method inherits. Verified by spec test (case 8). |
| T-03-12 | Tampering | Mass-assignment via UpdateVariantDto setting unknown fields (e.g., `id`, `createdAt`) | mitigate | Global `ValidationPipe({ whitelist: true })` from `main.ts` strips unknown fields. UpdateVariantDto via PartialType only allows title/description/published. |
| T-03-13 | Information Disclosure | Student fetches GET /variants/:id for an unpublished variant | accept | RESEARCH §Open Questions #2 recommendation: preview-by-id is shareable but not catalog-listed. listPublished filters published; getById does not. This is a known accepted design — students with a direct draft link can preview, but they cannot start an attempt (Phase 4 will gate attempt creation on `variant.published`). |
| T-03-14 | Information Disclosure | Validation error messages leak schema details (e.g., column names) | accept | NestJS `ValidationPipe` returns class-validator messages which include property names ("title must be a string"). This is acceptable for a self-service admin API. No PII or secret data leaks. |
| T-03-15 | Denial of Service | Unbounded pagination with `limit=999999` | mitigate | `Math.min(100, Math.max(1, parseInt(rawLimit, 10) || 20))` caps limit at 100. Applied to both admin and student list routes. |
| T-03-16 | Tampering | Path traversal via `id` parameter to access non-variant rows | mitigate | Prisma parameterized queries; `id` is treated as UUID by `findUnique({ where: { id } })`. No string interpolation. |
| T-03-17 | Repudiation | No audit log for admin variant publish/unpublish | accept | Phase 3 does not introduce audit logging. CONCERNS.md (project-wide) tracks this as a milestone-level gap; not in Phase 3 scope. |
</threat_model>
