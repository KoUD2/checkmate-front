<objective>
Build the variant builder at `/admin/variants/[id]`: a split-pane page (left = bank of all exam tasks; right = current variant tasks) with `@dnd-kit/sortable` drag-and-drop on the right, optimistic local state for all add/remove/reorder operations, and a single `Сохранить` button that posts the whole list atomically via `PUT /admin/variants/:id/tasks` (D-08).

Purpose: This is the centerpiece of Phase 3 — the surface where admins actually compose variants. It delivers VARIANT-02 (add tasks + drag-reorder). Inline title/description editing also delivers VARIANT-01 edit-of-meta after the create flow lands in the builder. The save bar publish toggle covers VARIANT-03 from within the builder context.

Output: 1 client component + 1 CSS module. The route folder `[id]/` did not exist before this plan; the page is created via Next.js dynamic route convention.
</objective>

`arrayMove(arr, fromIndex, toIndex)` returns a new array with the element moved. Use it inside `onDragEnd` handler.

`closestCenter` is the recommended collision detection for vertical lists.
